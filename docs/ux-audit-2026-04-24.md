# UX Audit — Energi Electric App
**Date:** 2026-04-24, post-rebrand wave 1
**Method:** Playwright walk-through of every authenticated page (admin + worker views) plus mobile spot-check. 17 screenshots captured at `docs/ux-audit-2026-04-24/`.
**Production URL:** https://energi-electric-app.vercel.app
**Build:** commit `1dc57fe` (auth pages polish merged)

## TL;DR

The auth pages (login, signup, forgot, reset) look great after PR #44. Everything **inside** the app — dashboard, projects, invoices, settings, reports, schedule, templates, activity — still uses the old Blue Shores palette throughout. **The Energi rebrand is ~30% visible to a real user.** Three big themes:

1. **Sidebar logo readability is broken** — same problem we fixed on auth pages. Dark gray sidebar + green logo wordmark = unreadable.
2. **Every accent color in the app is still Blue Shores green `#68BD45`** instead of Energi green `#045815`. That's covered by issues #9, #10, #11 — they're not "nice-to-have" polish, they're the bulk of the visible rebrand.
3. **Mobile has no logo at all** — bottom-tab nav replaces the sidebar but never substitutes a logo elsewhere.

## Severity rubric

| Tier | What it means |
|---|---|
| 🔴 P0 | Brand-broken or unreadable. Customer/Joe would notice immediately. |
| 🟡 P1 | Wrong color but functional. Wouldn't stop a launch but feels unfinished. |
| 🟢 P2 | Nice-to-have polish. |

## Findings by surface

### 🔴 P0 — Sidebar logo unreadable on every authenticated page

**Pages affected:** all 14 admin pages + 5 worker pages
**Screenshot:** `01-dashboard-desktop.png`, `06-projects-admin.png`, etc. (look at top-left)

The sidebar is dark gray (`#32373C`). The logo I staged (`/brand/energi-logo-horizontal.png`) is green wordmark on transparent. Against the dark gray, the wordmark looks like a smudge.

**Fix options:**
1. **Recommended:** generate a white-text version of the horizontal logo. Use it on dark backgrounds.
2. Lighten the sidebar background to white or near-white. Big visual change, would need to flip a lot of in-sidebar text colors.
3. Use the mark-only logo (`energi-logo-mark.png`) in the sidebar instead of the horizontal one. The mark-only is on a green circle with white E + white border — reads fine on dark gray.

Option 3 is the smallest change. Option 1 is the prettiest. Option 2 is the most work.

---

### 🔴 P0 — Mobile layout has no logo

**Screenshot:** `16-dashboard-mobile.png`

On mobile (390×844), the sidebar is replaced by a bottom-tab nav with: Home, Projects, My Time, Log Out. **The Energi logo is nowhere on screen.** Brand identity is invisible to phone users — which is most of the field crew.

**Fix:** add the Energi mark + wordmark to the mobile top header. Minimal — just stack the logo above the page title in the mobile breakpoint.

---

### 🔴 P0 — Mobile nav missing 6 sections

**Screenshot:** `16-dashboard-mobile.png`

Bottom-nav shows Home, Projects, My Time, Log Out (4 items). Desktop sidebar has 10 (Dashboard, Projects, My Time, Schedule, Reports, Invoices, Activity, Templates, Settings, Help). On mobile, **Schedule, Reports, Invoices, Activity, Templates, Settings, Help are unreachable** unless there's a hamburger I'm missing.

**Fix:** Either add a "More" tab that opens the rest, or a hamburger / drawer for the secondary items.

### 🟡 P1 — Every accent color in app is Blue Shores green

**Affected (non-exhaustive — this hits every page):**
- Active sidebar nav item (text + box highlight)
- Primary buttons: "New Project", "New Invoice", "New Template", "Mark as Sent", "Clock In"
- Progress bars (project completion)
- Status badges (Active, Paid, Sent, Draft, Scheduled, In Progress, Complete)
- Phase tabs (✓ Rough In, Trim Out, Final Walk)
- "View all", "Add new customer", "+ Add line item" links
- "Upload Blueprint", "Upload one now" links on project detail
- Sun/weather icons on dashboard
- Folder + clock + photo activity icons
- Section dividers / emphasis lines

All currently `#68BD45` (Blue Shores). All should be `#045815` (Energi).

**Tracked by:** Issues #9 (dashboard + projects), #10 (invoices + customers), #11 (settings + reports). The remaining surfaces (Schedule, Templates, Activity, project detail, project/new, invoice/new, my-time) aren't currently in any open issue. Inventory has all the file paths under "Bucket B / per-page brand application" — but actual issues only cover ~6 pages of the ~16 surfaces.

**Recommendation:** add 2-3 more issues to Operations to cover Templates, Schedule, Activity, project detail, and the various `/new` form pages. Or expand #9–#11 scopes.
